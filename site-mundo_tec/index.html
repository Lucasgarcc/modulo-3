Conceitos do HTML

HTML significa Hypertext Markup Language ou Linguagem de Hipertexto por Anotações e não é uma linguegem de programação (Linguagem de marcação de texto ), é uma implementação simultânea da norma ISO 8879:1986 SGML (Standard Generalized Markup language) e da norma ISO 10744:1992 HyTime (Hypermedia/Time-based Document Structuring Language. Embora não tenha sido concebida para ser aplicada a hipertextos, a SGML define um padrão segundo o qual documentos tradicionais podem ser transformados em hyper-objetos. Já o HyTime é um padrão que define formas de representação e estruturação de hipermídia, como também qualquer informação temporal.

Documentos HTML 5 podem ser interpretados por navegadores. A tecnologia que pode ser implementado conteúdos textos, imagens, videos e audios.

html é o ficheiro (arquivo) que em muitos servidores web é servido por default (padrão) caso o URL requisitado corresponda a um directório. Geralmente é o primeiro ficheiro (arquivo) de um web site

Tags é um código usado para marcar o início e, onde for requerido, o fim de um elemento HTML. Há, como exposto acima, tags de abertura e de fechamento.

HTML: Linguagem de Marcação de Hipertexto

As páginas da web podem e serão muito diferentes umas das outras, mas todas tendem a compartilhar componentes padrão semelhantes, a menos que a página exiba um vídeo ou um jogo em tela cheia, seja parte de algum tipo de projeto de arte ou seja mal estruturada:

cabeçalho (header)

Normalmente, uma grande faixa na parte superior com um grande título e / ou logotipo. É aí que as principais informações comuns sobre um site geralmente ficam de uma página para outra.

barra de navegação

Links para as principais seções do site; geralmente representado por botões de menu, links ou guias. Como o cabeçalho, esse conteúdo geralmente permanece consistente de uma página para outra - ter uma navegação inconsistente em seu site levará a usuários confusos e frustrados. Muitos web designers consideram a barra de navegação parte do cabeçalho em vez de um componente individual, mas isso não é um requisito; na verdade, alguns também argumentam que ter os dois separados é melhor para acessibilidade, já que os leitores de tela podem ler melhor os dois recursos se estiverem separados.

conteúdo principal

Uma grande área no centro que contém a maior parte do conteúdo exclusivo de uma determinada página da Web, por exemplo, o vídeo que você deseja assistir ou a história principal que você está lendo ou o mapa que deseja visualizar ou as manchetes de notícias, etc. Esta é a única parte do site que definitivamente irá variar de página para página!

barra lateral (sidebar) Algumas informações periféricas, links, cotações, anúncios etc. Geralmente, isso é contextual ao conteúdo principal (por exemplo, em uma página de um artigo de notícias, a barra lateral pode conter a biografia do autor ou links para artigos relacionados), mas há também casos em que você encontrará alguns elementos recorrentes como um sistema de navegação secundário.

rodapé (footer)

Uma faixa na parte inferior da página que geralmente contém letras pequenas, avisos de direitos autorais ou informações de contato. É um lugar para colocar informações comuns (como o cabeçalho), mas geralmente essas informações não são críticas ou secundárias ao próprio site. O rodapé também é usado às vezes para fins de SEO, fornecendo links para acesso rápido a conteúdo popular. Um "site típico" poderia ser colocado assim:

ATRIBUTOS: "Elelementos"

Atributos são informações que passamos na Tag para que ela se comporte da maneira esperada.

Atributos Universais:

As classes são uma forma de identificar um grupo de elementos. Através delas, pode-se atribuir formatação a VÁRIOS elementos de uma vez. Exemplo:

div class: "classe1"

Depois de adcionar uma div class no seu html, você consiguirá editar os elementos, apenas selecionando aquela classe, pois ela vai estar redirecionada aos elementos.

As ids são uma forma de identificar um elemento, e devem ser ÚNICAS para cada elemento. Ao acrescentar um id no seu html, poderá seleciona-lo indicando o id em que tal está redirecionado Vemos assim que a diferença é que um id é único e o class é mais geral.

TAGS MAIS USADAS

  • A declaração "!DOCTYPE" deve ser a primeira coisa no seu documento HTML, antes da tag "html".
  • A tag a define um hiperlink, que é usado para vincular de uma página para outra.
  • A tag article especifica conteúdo independente, separando então em artigos
  • A tag audio serve para colocar um audio na página ;)
  • A tag body define o corpo do documento.
  • A tag captiondefine uma legenda da tabela.
  • A tag del define o texto que foi excluído de um documento.
  • A tag div define uma divisão ou seção em um documento HTML.
  • A tag em serve para enfatizar o texto
  • A marca font especifica a face da fonte, o tamanho da fonte e a cor do texto. Temos então alguns tipos (muito USADAS dentro do CSS- estiliza texto ) font family: estilo da letra font size: tamanho font color : cor
  • A tag footer define um rodapé para um documento ou seção
  • As tags h1 h1 > até o h6 são usadas para definir títulos HTML.
  • O elemento head é um contêiner para todos os elementos principais.
  • A tag ul define uma lista não ordenada (com marcadores), e ol uma lista ordenada. Ambas usam a tag li para representar um item da lista
  • A tag title é necessária em todos os documentos HTML e define o título do documento.
  • A tag strong deixa o texto em negrito
  • A tag select define seções em um documento, como capítulos, cabeçalhos, rodapés ou quaisquer outras seções do documento.
  • A tag p define um parágrafo.
  • A tag nav define um conjunto de links de navegação.
  • A tag mark define o texto marcado.
  • A tag img define uma imagem em uma página HTML.
  • A tag table determina a criação de uma tabela
  • A tag tbody define o corpo da tabela
  • A tag define o rodapé da tabela
  • A tag td as células da tabela
  • A tag th representa a célula de cabeçalho de uma tabela
  • A tag thead representa o agrupamento do conteúdo de uma tabela A tag serve para acrescentar links
